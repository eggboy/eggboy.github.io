Assumptions & Methodology
  • Weight precision: FP16 (2 bytes per parameter) for dense models. GPT-oss MoE models use MXFP4 quantization (0.5 bytes) for expert weights + BF16 for other weights — actual model sizes are provided as overrides (e.g. GPT-oss-120B ≈ 63 GB, fits a single H100).
  • KV cache precision: FP16 (2 bytes per value). Size per token = 2 × layers × kv_heads × head_dim × 2 bytes.
  • Memory footprint = model weights + KV cache for all concurrent requests across the full context window (input + output).
  • Mixture-of-Experts (MoE) models MoE: Some models (like GPT-oss) use a technique where only a small fraction of the total weights are used per token. All weights must still fit in GPU memory, but per-token compute is much faster — we use the "active parameters" for latency calculations and "total parameters" for memory.
  • Prefill time/token (compute-bound): 2 × active_params / GPU_TFLOPS. This is the time to process each input token.
  • TPOT — Time Per Output Token (memory-bandwidth-bound): 2 × active_params / bandwidth × 1000 ms.
  • TTFT — Time To First Token ≈ prefill_time + TPOT/1000.
  • E2E Latency = (input × prefill_time + output × TPOT) / 1000 s.
  • Throughput = output_tokens / E2E_latency (single-request output token rate).
  • GPU count is fixed at 1 GPU. For multi-GPU setups, memory and compute scale linearly (use the Python CLI for multi-GPU).
  • These are theoretical best-case estimates. Real-world performance depends on serving framework, batching, quantization, and system overhead.
